SHIVA honeypot: HELP

How to work with honeypot

On the main page is presented overview of last emails caught by honeypot. It's score and classification result is also presented.

You can view details of email by clicking on it's id, delete it from honeypot by clicking on the or mark is as spam/phishing by clicking on or


Sensor

Sensor represents source of email.

sensor description
shiva email was caught by honeypot
.*spamImport.* email was manually imported to honeypot as spam
.*phishingImportImport.* email was manually imported to honeypot as phishing

Scores

Honeypot computes two scores. Each score represents probability that email is phishing.

Shiva score is result of phishing detection technique implemented in honeypot.
Spamassassin score is result of Bayes classifier in Spamassassin.

IMPORTANT imported emails (sensors .*spamImport.*,.*phishingImport.*) have always default score -1, they are used as learning data only


Learning honeypot

After instalation, hoenypot is learnt on the provided data samples. This process is described in package readme.txt file

When honeypot catches emails, it's possible to relearn it. It will take into considerations cought emails as well.

By default, spammassassin boundary for for marking email is 0.5. This boundary can be shifted by re-learning to number ininterval <0.4,0.6>
